# Repository Notes

This repository is structured for agent-assisted development with a simple harness.

Priority order for local work:

1. `AGENTS.md`
2. `harness/session-checklist.md`
3. files under `state/`
4. the current task or feature spec

Repository expectations:

- Product design belongs outside this scaffold until explicitly added.
- `apps/` contains runnable applications.
- `packages/` contains shared contracts and reusable modules.
- `tests/` contains repository-level validation and smoke coverage.
- `docs/development/` contains workflow documentation only.

When adding real implementation later, keep feature scope explicit and verifiable.
